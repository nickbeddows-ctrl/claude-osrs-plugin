Available tools

Everything your AI needs to know

The plugin exposes your character data as MCP tools. Your AI calls them on demand — it never has access to anything you haven't explicitly allowed.

get_all
All player data in a single call — the fastest way to give your AI full context.
get_player_stats
All 23 skill levels, XP, XP to next level, and overall combat level.
get_equipment
Every equipped item by slot: head, cape, neck, weapon, body, offhand, legs, gloves, boots, ring, ammo.
get_inventory
Full inventory contents with item names and stack quantities.
get_location
World coordinates plus a human-readable area name (e.g. "Slayer Tower", "Grand Exchange").
get_quest_states
All quests split into completed, in progress, and not started — plus total quest points.
get_diary_states
All 12 Achievement Diary regions with easy, medium, hard and elite completion status.
get_slayer_task
Current Slayer task: creature, kills remaining, location, points and streak.
get_clue_scroll
Whether you have an active clue scroll and which tier it is.
get_ge_offers
All active Grand Exchange offers: item, quantity, price per item and current state.
Connection modes

Connect however works best for you

Start with Local if your AI tool is on the same machine. Use LAN, Tailscale or Cloud Relay if you're connecting from a different device.

1
Local Recommended
RuneLite and your AI tool on the same machine. Default mode — no extra setup beyond adding the config snippet to your AI tool.
2
LAN
RuneLite on one device, AI tool on another — both on the same network. The plugin panel shows your LAN IP and the exact config snippet to copy.
3
Tailscale Best for cross-device
Devices on different networks. Install Tailscale (free) on both, sign in with the same account, and the plugin walks you through the rest. Stable permanent IP.
4
Cloud Relay No install needed
SSH reverse tunnel via serveo.net — SSH is built into macOS and Windows 10+. Creates a public HTTPS URL with no software to install. URL changes on restart unless you set up a stable subdomain.
Quick setup

Up and running in 3 steps

These steps are for Local mode with Claude Desktop. See the README for other AI tools and connection modes.

1
Install the plugin
Search for OSRS MCP in the RuneLite Plugin Hub and click install. Once installed, enable it from the sidebar.
2
Add to Claude Desktop config
Open your Claude Desktop config file — the plugin panel shows the exact path — and add the following. Or use the Copy config button in the plugin panel.
{
  "mcpServers": {
    "osrs": {
      "command": "npx",
      "args": ["mcp-remote", "http://127.0.0.1:8282/mcp"]
    }
  }
}
Config file locations:
macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows: %APPDATA%\Claude\claude_desktop_config.json
3
Restart Claude Desktop and ask away
Quit and reopen Claude Desktop. Log into RuneLite, then ask Claude anything about your character. It will call the relevant tools automatically.
Other AI tools
The plugin works with any MCP-compatible tool. Point it at http://127.0.0.1:8282/mcp using streamable HTTP transport, or use npx mcp-remote as a stdio bridge.

Cursor, Windsurf, and other tools are covered in the README.
Privacy

You control what the AI can see

Every data category has an individual toggle in plugin settings. The server is read-only and never sends commands to the game.
